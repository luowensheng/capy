# Handlers package

Generated by Capy from `script.capy`. 4
handlers declared:

- `GET /users` → `ListUsers` returns `[]User`
- `GET /users/{id}` → `GetUser` returns `User`
- `POST /users` → `CreateUser` returns `User` (accepts `UserCreateRequest`)
- `DELETE /users/{id}` → `DeleteUser` returns `void`

## How to add a handler

1. Add a `handler ... method ... path ... returns ...` line to
   `script.capy` (one line).
2. Run `capy run --out-dir . lib.capy script.capy`. The handler
   stub appears in `internal/handlers/handlers.go`, the test in
   `handlers_test.go`, and this README updates.
3. Implement the handler body. The smoke test will start failing
   (because the response is no longer 501); replace it with a
   real assertion of your contract.

The directory layout is fixed by the library: never argue about
where handlers live again.
